Introduction

The configuration of servers in getcal is the same than in ics2qtcal.
The main idea is to get ics files (they can end by .ics or not) from a server using some parameters.

Configuration

Name is a unique name to save these settings

Server tab

Server URL

This is the place to save your calendar URL.
When you set this setting, you set the way you'll use the Calendars tab :

  • If you have only one calendar on this server, you can specify its full address in Server URL parameter and let the Calendars part empty.
  • If you have several calendar files hosted on the same server, you'll have a big part of their URL in common.
    In this case, you can set the common URL part in Server URL and add the non-common part of each calendar URL in the Calendars tab.

Username and Password

To download calendar files, you may need to enter a Username and a Password.
If you need them, save them in the right fields. Otherwise, let these two fields empty.

Calendars tab

Depending on your situation, you can let this part empty, see Server URL configuration.

If you need to add different calendars, you can add here every one by adding an entry in the list with the Add calendar button (you have to set a text on the left of the button and then click on it).

Examples

One user with one calendar

In this case, you have only to fill the Name and Server URL settings (and if needed the Username and Password

  1. Name : a unique name
  2. Server URL : http://example.org/path/to/my/calendar
  3. Username : Toto
  4. Password : ********

One user with several calendars

For example, if you use the Davical server on example.org you can have some calendars with this urls :

  1. http://cal.example.org/davical.php/user1/Personnel
  2. http://cal.example.org/davical.php/user1/Job.ics
  3. http://cal.example.org/davical.php/user1/Birthdays

In this case you'll write "http://cal.example.org/davical.php/user1" in Server URL and you'll add calendars :

  1. Personnel
  2. Job.ics
  3. Birthdays

One server with two users sharing some calendars

Let say that there's another user, user2, which have some calendars on the same Davical server and that user1 can access user2's calendars within the server.
In this case, instead of creating two servers with same user and password, you can set server URL parameter as this : "http://cal.example.org/davical.php/" and add calendars :

  1. user1/Personnel (so the end of URL)
  2. user1/Job.ics
  3. user1/Birthdays
  4. user2/Birthdays (as it's the end of URL and user1 have access to user2's calendars)
  5. user2/Personnel
